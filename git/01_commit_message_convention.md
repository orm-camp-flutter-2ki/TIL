# Commit Message Convention

> 커밋 메시지 컨벤션

```커밋 타입: 커밋 내용```
형태로 작성

### 커밋 타입 종류
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정
- style: 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
- refactor: 코드 래픽토링
- test: 테스트 코드, 리펙토링 테스트 코드 추가
- chore: 빌드 업무 수정, 패키지 매니저 수정

### 커밋 내용 작성 요령
- 쉽고 간결하게 작성하기
- 과거시제 사용하지 않기
- 일관적인 형태로 작성하기

### Example
```feat: create signup model```
<br>
```fix: image bug fix```
